# 项目与分镜审核

> 长久记忆：项目卡片如何复用全局工作流并进入分镜审核。

## 创建与运行

项目页以卡片网格展示名称、所选工作流、运行状态与更新时间。新建项目先填写项目名并选择一份全局工作流模板，再进入与工作流模块相同的运行输入页。文本可直接粘贴，也可导入 TXT、Markdown、RTF、含文本 PDF 或 DOCX 后继续编辑；最多 80,000 字符，无文本 PDF 明确提示“不支持 OCR”。提示词和知识集合可沿用默认值，图片、视频、音频或文件夹等本次素材由用户在真正开始运行前选择。未填写运行输入不会被误判为工作流配置错误。

用户确认“开始运行”后才创建项目，``WorkflowExecutor`` 立即返回稳定的运行 ID，``ProjectStore`` 将其写入项目记录。工作流定义和媒体不会复制到项目目录；定义继续来自全局模板，运行记录与产物继续保存在 `Workflows/Runs/`。项目重新运行时也会重新打开运行输入页，不会沿用已经失效的本地素材路径。

汽车内广告工作流缺资料时，项目进入“待补资料”而不是失败。补库子运行属于全局“添加知识库”历史，项目始终只绑定父广告运行。子运行完成后父运行自动重检并恢复；关闭向导、子运行失败或重启应用不会丢失缺口，用户可在项目详情继续补库或取消项目。

## 自动进入审核

运行成功或带警告完成时，项目从工作流的结果输出节点提取文本，并按“镜头 / 镜号 / 分镜 / 数字列表”等常见 Markdown 格式容错拆成单镜头卡片。工作流节点产出的图片和视频按稳定顺序分配给卡片。汽车工作流的“参考资料”内部元数据会被移除，再按稳定 ID 映射 `Assets/KnowledgeReferences/` 中的知识原图；遗漏或无效 ID 使用本次有效参考包兜底并写入项目警告。若工作流没有文本输出，审核页仍会打开并允许手动添加分镜。项目状态进入“待审核”，界面自动导航到分镜审核页。

每张卡片横向展示混合参考图、镜头共用提示词和 9:16 视频。参考图支持原比例预览、最多 9 张缩略图、文件导入、直接生图和解除关联；提示词防抖自动保存；视频区支持生成前确认服务商、模型、比例、分辨率、时长和声音选项，并显示生成进度、取消状态与本地 MP4 播放。没有视频时显示应用图标。媒体生成复用全局适配器和钥匙串密钥，不静默发起付费调用。

审核页仍支持填写审核意见、保存修改和确认通过。修改已通过项目会使其回到“待审核”；再次确认后恢复“已通过”。运行失败或取消的项目保留错误信息，并可使用同一工作流重新运行。

## 持久化边界

``ProjectRecord`` 保存项目名、工作流 ID 与名称快照、运行 ID、状态、兼容旧版的整段分镜草稿、卡片式镜头、媒体相对路径、审核意见、警告和时间。``ProjectStore`` 将全部项目原子保存到 `Projects/project-board.json`；旧项目首次加载时自动补齐卡片并保存。导入和生成媒体只写入 `Workflows/Runs/<workflowID>/<runID>/Assets/`。删除卡片或项目不会删除关联的全局运行历史和媒体。
